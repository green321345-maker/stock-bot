# Stock Bot — 신뢰도 중심 주식 모니터링 앱 설계

이 저장소는 다음 목표를 위한 **제품/기술 설계 초안**을 담고 있습니다.

- 주식 모니터링, 검색, 고급 필터링
- 매수 타이밍 알림
- 복수 투자 철학 기반 점수화 (워런 버핏, 피터 린치 등)
- 예상 가격(밸류에이션/시나리오)
- 정확도·신뢰도 중심의 데이터 품질 관리

## 문서 구성

- `docs/prd.md`: 기능 요구사항 및 MVP 우선순위
- `docs/system-architecture.md`: 시스템 아키텍처
- `docs/scoring-model.md`: 투자 철학 기반 점수/신뢰도 모델
- `docs/ui-ux-guidelines.md`: 이전 앱의 디자인 불만족 개선을 위한 UX 원칙

## 권장 개발 순서 (12주)

1. 주차 1~2: 데이터 소스 확정, 티커/재무 DB 스키마 구축
2. 주차 3~4: 검색·필터링 API + 기본 대시보드
3. 주차 5~6: 버핏/린치 점수 엔진 v1
4. 주차 7~8: 알림 엔진(조건식 + 우선순위)
5. 주차 9~10: 예상 가격 엔진(DCF + 멀티플)
6. 주차 11~12: 백테스트/모니터링/모델 리포트

## 핵심 원칙

- 해석 가능한 점수(Why 설명 가능)
- 데이터 신뢰성(출처/타임스탬프/결측 처리)
- 과최적화 방지(워크포워드 검증)
- 투자 조언이 아닌 의사결정 지원 도구


## 기업 데이터 출처 (신뢰도 우선)

- **공시 원문(1차 소스)**: SEC EDGAR(미국), DART(한국)
- **시장 데이터**: 거래소/공식 벤더의 EOD/분봉 데이터
- **보조 벤더(2차 소스)**: 재무 정규화/과거 시계열 제공 벤더
- **뉴스/이벤트**: 공시, 실적발표 캘린더, 신뢰 가능한 뉴스 피드

> 원칙: 같은 지표를 최소 2개 소스로 교차 검증하고, 불일치 시 신뢰도 점수를 자동 하향합니다.


## 추가 반영 요청

- **실시간 알림**: 매수/매도 조건이 충족되면 앱 푸시, 이메일, 웹훅으로 즉시 알림
- **디자인 톤**: 기본 중립 톤에 "살짝 보라빛" 포인트 컬러 적용
