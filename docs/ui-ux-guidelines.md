# UI/UX 가이드 ("이전에 만든 앱이 마음에 들지 않았다" 문제 해결)

## 1) 정보 우선순위

1. 지금 사도 되는가? (점수 + 신뢰도 + 위험)
2. 왜 그런가? (근거 3~5개)
3. 언제 알림이 오나? (조건 + 상태)

## 2) 화면 구조

- Home: 시장 개요 + 내 Watchlist
- Screener: 고급 필터 빌더 + 결과 테이블
- Stock Detail:
  - 상단: 가격/변동/요약
  - 중단: 전략별 점수 카드
  - 하단: 예상 가격 시나리오 + 리스크
- Alerts: 규칙/히스토리/우선순위

## 3) 디자인 원칙

- "어두운 배경 + 고채도" 남용 금지
- 카드 간격 8pt 그리드 통일
- KPI 숫자 크기 계층 명확화
- 한 화면에서 색상 의미는 3개 이내(상승/하락/중립)

## 4) 신뢰 UX

- 각 지표 옆에 정보 아이콘: 정의/출처/업데이트 시각
- "모름"을 숨기지 않고 표시
- 예상 가격은 단일 숫자 대신 범위 + 가정 노출

## 5) 알림 UX

- 알림 원인 요약 1줄 + 상세 근거 펼침
- 동일 원인의 반복 알림 묶음 처리
- 사용자별 알림 빈도 상한


## 6) 컬러 시스템 (은은한 보라 포인트)

- Primary Accent: `#7C4DFF`
- Secondary Accent: `#9B7BFF`
- Background: `#0F1115` 또는 `#FFFFFF`(라이트 모드)
- 상승/하락 색상은 기존 금융 관습 유지(녹/적), 보라는 CTA/선택 상태에만 사용

적용 위치:
- 선택된 탭, 주요 버튼, 하이라이트 카드 테두리, 진행 상태 배지
