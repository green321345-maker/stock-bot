# PRD (Product Requirements Document)

## 1) 제품 목표

투자자가 여러 투자 대가의 검증된 프레임워크를 한 앱에서 비교/활용하도록 돕고,
**신뢰도 있는 데이터와 설명 가능한 점수**로 매수 의사결정을 지원한다.

## 2) 핵심 사용자

- 가치투자 성향 개인 투자자
- 종목 모니터링 자동화가 필요한 투자자
- 퀀트 지표를 쓰되 해석 가능성을 원하는 사용자

## 3) MVP 핵심 기능

### A. 모니터링 대시보드
- 관심종목(Watchlist) 관리
- 실시간/지연 시세, 거래대금, 변동률, 재무 요약 카드
- 뉴스·공시 이벤트 타임라인

### B. 검색/필터링
- 재무 필터: PER, PBR, ROE, 부채비율, FCF, 매출성장률
- 품질 필터: 이익 안정성, 마진 추세, 내부자 매수/매도
- 조건 저장/공유(예: "버핏 저평가 후보")

### C. 점수 엔진
- Buffett Score (가치+해자+재무건전성)
- Lynch Score (성장+PEG+이해가능성)
- Quality Score, Risk Score, Composite Score

### D. 매수/매도 타이밍 실시간 알림
- 조건식 기반 알림: 가격 하락/지표 개선/실적 발표/기술적 시그널
- 실시간 트리거: 초/분 단위 가격 스트림 + 룰엔진 평가
- 채널: 앱 푸시, 이메일, 웹훅(텔레그램/슬랙 연동 가능)
- 중요도 점수와 알림 피로도 방지(쿨다운)
- 알림 본문에 "왜 발생했는지"(근거 3개 이내) 자동 첨부

### E. 예상 가격
- DCF 기반 내재가치 범위
- 멀티플(역사적 PER/PBR, 동종업계 밴드)
- 베어/베이스/불 3시나리오

## 4) Non-Functional 요구사항

- 정확도: 데이터 검증 파이프라인 + 이상치 탐지
- 신뢰도: 지표별 출처/기준일 표시
- 설명성: 점수 산출 근거를 카드형으로 노출
- 확장성: 전략 플러그인 구조
- 보안: API 키 보관, 최소권한 원칙

## 5) 성공 지표

- 30일 유지율, 알림 클릭률, 관심종목 재방문률
- 알림 후 사용자 액션 전환율
- 백테스트 샤프/최대낙폭 및 거래비용 반영 성능
- 사용자 신뢰 지표(설명 이해도 설문)

## 6) 제외 범위 (초기)

- 초단타 고빈도 트레이딩
- 레버리지/옵션 자동매매
- 투자자문 행위


## 7) 데이터 소스 정책 (기업 자료는 어디서 가져오나)

### 1차 소스(최우선)
- 미국 상장사: **SEC EDGAR** 10-K/10-Q/8-K
- 한국 상장사: **DART** 사업보고서/분기보고서
- 배당/액면분할/자사주 등 기업행위: 거래소/공시 원문

### 2차 소스(정규화/속도 보완)
- 시세/재무 데이터 벤더 API(라이선스 검토 필수)
- 뉴스/실적 캘린더 API

### 품질 규칙
- 지표마다 `source_type(primary/secondary)`, `source_name`, `as_of_date`, `ingested_at` 저장
- 1차 vs 2차 값 오차가 임계치 초과 시 "검증 필요" 플래그
- 결측/지연 데이터는 점수 산출에서 신뢰도 감점 처리


## 8) UI 브랜딩 요구사항

- 기본 색상 체계는 가독성 중심 중립 톤 유지
- 포인트 컬러는 **은은한 보라 계열** 사용 (예: `#7C4DFF`, `#9B7BFF`)
- 상승/하락 색상(녹/적)과 보라 포인트의 역할을 분리해 혼동 방지
